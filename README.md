### 发送协议
整体的协议看起来是`0x53 0x5A 0x48 0x59 cmd theta R sum`  
发送协议共包含XBytes，其中 4 Bytes 为包头`0x53 0x5A 0x48 0x59`，1 Byte 为控制的动作，定义如下表：
|数据|含义|
|---|---|
|`0x01`|抓取球|
|`0x02`|抓取魔方|
|`0x03`|投放球|
|`0x04`|投放魔方|
|`0x05`|四两拨千斤|
|`0x06`|爪子复位（可以同时移动机械臂）|
|`0x07`|爪臂上移（可以同时移动机械臂）|
|`0x08`|爪臂下移（可以同时移动机械臂）|
|`0x09`|抓边沿球|
|`0x0A`|抓边沿魔方|
|`0x0B`|上提机械臂|
|`0x0C`|单放爪，爪张开|
再有 2 Bytes 控制机械臂位置，前一 Byte 是回转电机角度值$\theta$，范围为 -180°~180° ，由度数转化为数据的公式为：
当转角$\theta$在[0°, 180°]时：
$$数据 = int(\theta)$$
当$\theta$在[-180°, 0°]时：
$$数据 = int(\theta + 180°)$$
由于取值只有181种可能，故用 1 byte 来表示。  
对于直线电机的值，计算公式如下
$$
\left\{\begin{matrix}
 int\left ( \frac{127}{30}R \right ), \theta \ge  0
 \\
 int\left ( -\frac{127}{30}R \right ), \theta \le   0
\end{matrix}\right.
$$
这实际上时用 7 bits 表示了极径，借用了符号位来表示转角那一位的正负，整体上和极坐标定义一致，但要注意发送的数据由于是负数，实际上发的是其补码。
## UI
- 蓝色十字准星：世界系原点在地面上的投影点
- 蓝色外圆：盒子抓取范围
- 红色内圆：球抓取范围
- 左键点击：（manual）抓取目标位置
- 右键点击：（place）放置目标位置
- 电控复位：把爪移到中心，利于飞行时保持平衡
